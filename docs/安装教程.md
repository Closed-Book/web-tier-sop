# 安装教程（中文详版）

本文覆盖：**环境依赖** → **下载与安放位置** → **需按自己机器修改的地址** → **启动 watchdog** → **验证**。

> 仅支持 **macOS**。主 Chrome 9222 watchdog 与 Tier 3 应急资产都依赖 macOS launchd 机制。

> 📌 **2026-05-25 架构合并**：原三层已合并为**两层**（Tier 1 `WebFetch` / Tier 2 主 Chrome 9222 后台 tab），所有反爬 + 登录态任务（含 X / Twitter）统一走 Tier 2。原 Tier 3 独立浏览器（Brave 9223）**已下线**，改为 opt-in 应急资产（`--with-tier3`）。历史与复活流程见 [`tier3-rollback.md`](tier3-rollback.md)。

## 一、环境依赖

### 1.1 必需依赖（不装跑不起来）

| 依赖 | 要求 | 检查命令 | 安装方式 |
|---|---|---|---|
| **macOS** | —— | `sw_vers` | 系统 |
| **Claude Code 或 Codex** | 最新版 | —— | 宿主环境，本 SOP 通过 skill 触发 |
| **Node.js** | **≥ 22**（opencli 用；Tier 3 应急 `helper.mjs` 用原生 `WebSocket` 也需 22） | `node -v` | [nodejs.org](https://nodejs.org) / `brew install node` / nvm |
| **npm** | 随 Node 自带 | `npm -v` | —— |
| **Google Chrome** | 任意近版 | `ls "/Applications/Google Chrome.app"` | [google.com/chrome](https://www.google.com/chrome/) |

> Google Chrome 是 Tier 2 唯一链路的载体：用户日常主 Chrome 带 `--remote-debugging-port=9222` 启动，承载所有反爬 + 登录态任务。

### 1.2 可选依赖（按用到的功能装）

| 依赖 | 用途 | 不装的后果 | 安装方式 |
|---|---|---|---|
| **web-access 插件** | Tier 2 路由（唯一反爬 + 登录态链路） | 只剩 Tier 1，路由表 Tier 2 分支缺位 | 见下方「二、3」 |
| **lark-cli** | watchdog / 失效处理发飞书 IM 告警 | 告警只写本地日志，不报错 | 你的飞书 CLI 工具 |
| **jq** | 运行 `references/site-patterns/` 里的抓取示例脚本 | 示例脚本里的 `jq` 行会失败（自己改写即可） | `brew install jq` |
| **Brave Browser** | **仅 Tier 3 应急复活时**（`--with-tier3`）需要 | 不复活 Tier 3 用不到 | `brew install --cask brave-browser` |
| **Tailscale** | **仅 Tier 3 应急**远程 SSH 救回登录态 | 两层默认用不到 | [tailscale.com](https://tailscale.com) |

### 1.3 系统自带（macOS 默认有，无需安装）

`curl`、`lsof`、`sqlite3`、`perl` —— watchdog / opencli 超时护栏 / Tier 3 应急脚本会用到，macOS 默认都带。

### 1.4 自动安装（`install.sh` 负责，无需手动）

| 依赖 | 说明 |
|---|---|
| **@jackwener/opencli** | 锁定 **1.7.22**。`install.sh` 跑 `npm ci` 装进 `skills/opencli-web/node_modules/`（该目录不入库，从 `package-lock.json` 重建） |

## 二、下载与安放位置

### 1. 下载本仓库

放哪都行（仓库会被 `install.sh` 软链到 skill 目录，不要装好后再移动它）：

```bash
git clone <本仓库地址> ~/web-tier-sop
cd ~/web-tier-sop
```

### 2. 安装本仓的两个 skill

**方式 A：自动（推荐）**

```bash
./install.sh                 # 默认等价于 --host claude，装到 ~/.claude/skills/
./install.sh --host codex    # Codex 用户，装到 ~/.agents/skills/
./install.sh --host both     # Claude Code 与 Codex 都装
```

`install.sh`（默认两层）会：按宿主软链 `skills/web-tier`、`skills/opencli-web`、`npm ci` 装 opencli、拷 `watchdog-9222.sh` 到 `~/.web-tier/`（渲染注入告警 open_id）、渲染 watchdog plist 到 `~/Library/LaunchAgents/`。

> 加 `--with-tier3` 会**额外**安装 Tier 3 独立 Brave 应急复活资产（`launch.sh` / `health.sh` + `.plist.disabled`），平时不用，详见 [`tier3-rollback.md`](tier3-rollback.md)。

**方式 B：手动**

Claude Code（Codex 把 `~/.claude/skills` 换成 `~/.agents/skills`）：

```bash
# skill 必须位于 <skill 目录>/<skill名>/，宿主才扫描得到
ln -s ~/web-tier-sop/skills/web-tier     ~/.claude/skills/web-tier
ln -s ~/web-tier-sop/skills/opencli-web  ~/.claude/skills/opencli-web
# opencli 依赖
cd ~/web-tier-sop/skills/opencli-web && npm ci
# watchdog 脚本（渲染告警 open_id：未设环境变量则留空，告警静默失败到 log）
mkdir -p ~/.web-tier
sed "s|__OPEN_ID__|${WEB_TIER_ALERT_OPEN_ID:-}|g" \
  ~/web-tier-sop/runtime/watchdog-9222.sh > ~/.web-tier/watchdog-9222.sh
chmod +x ~/.web-tier/watchdog-9222.sh
# watchdog launchd plist：把模板里的 __HOME__ 换成你的家目录
sed "s|__HOME__|$HOME|g" \
  ~/web-tier-sop/runtime/com.user.chrome-9222-watchdog.plist.template \
  > ~/Library/LaunchAgents/com.user.chrome-9222-watchdog.plist
```

> **关键**：Claude Code 的 skill 目录是 `~/.claude/skills/<skill名>/SKILL.md`，Codex 是 `~/.agents/skills/<skill名>/SKILL.md`，不能放别处。

### 3. 下载 web-access 插件（Tier 2，第三方，可选）

`web-access` 是第三方插件（作者「一泽 Eze」），**不在本仓**，需单独下载。

**方式 A：Claude Code 插件命令（推荐）**

在 Claude Code 里执行：

```
/plugin marketplace add https://github.com/eze-is/web-access.git
/plugin install web-access
```

装好后它位于 `~/.claude/plugins/cache/web-access/`，由 Claude Code 自动管理，**无需手动安放**。

**方式 B：手动 git clone**

```bash
git clone https://github.com/eze-is/web-access.git
# 之后按 web-access 仓库自己的 README 安放
```

> **关于「更改地址」**：上面的 marketplace 地址 `https://github.com/eze-is/web-access.git` 是当前来源。
> 若原作者迁移仓库换了地址，把这个 URL 换成新地址即可，其余步骤不变。
>
> 不装 `web-access` 也能跑 —— 只是只剩 Tier 1，路由表少了 Tier 2 分支（反爬 + 登录态任务无链路）。

## 三、需按自己机器修改的地址 / 配置

| 配置项 | 默认值 | 何时需要改 | 改哪里 |
|---|---|---|---|
| 家目录路径 | `install.sh` 自动渲染 | **仅手动安装时** —— 把 plist 模板的 `__HOME__` 换成 `$HOME` | `~/Library/LaunchAgents/com.user.chrome-9222-watchdog.plist` |
| 飞书告警目标 | 空（仅写本地日志） | 想收 watchdog 飞书 IM 告警 | 装前 `export WEB_TIER_ALERT_OPEN_ID=ou_xxxx`（用 `lark-cli auth list` 拿你的 open_id），`install.sh` 会渲染进 watchdog |
| web-access 来源地址 | `https://github.com/eze-is/web-access.git` | 原作者迁仓 | 上方「二、3」的 marketplace 命令 |
| **（Tier 3 应急）** 独立浏览器端口 | `9223` | 仅 `--with-tier3` 且端口被占用（少见） | `launch.sh` `PORT` + `web-tier-chrome.plist.template` `--remote-debugging-port` |

## 四、启动主 Chrome 9222 健康监控（两层主流程）

两层架构无需预热独立浏览器，主流程是：让日常主 Chrome 带 9222 调试端口，再启动 watchdog 兜底监控。

1. **确认主 Chrome 启动时带 `--remote-debugging-port=9222`**，并已登录你日常用的账号（X / 微博 / 知乎等）。
2. **启动 watchdog 守护**（持续 DOWN ≥ 5min 才飞书告警，**不自动拉起** Chrome，避免和日常 Cmd+Q 冲突）：

```bash
launchctl bootstrap gui/$(id -u) ~/Library/LaunchAgents/com.user.chrome-9222-watchdog.plist
```

> **（可选）Tier 3 独立浏览器应急复活**：仅当主 Chrome 9222 抓 X 在实战中连续失败 ≥ 3 次、且你明确要复活时，才走 `./install.sh --with-tier3` + 独立 Brave 预热流程。完整步骤见 [`tier3-rollback.md`](tier3-rollback.md) 和 [`../skills/web-tier/README.md`](../skills/web-tier/README.md)。

## 六、验证安装

```bash
~/.web-tier/watchdog-9222.sh                          # 手动跑一次，写入 ~/.web-tier/9222-watchdog.log
lsof -nP -iTCP:9222 -sTCP:LISTEN                       # 主 Chrome 带 9222 启动后应有监听
launchctl print "gui/$(id -u)/com.user.chrome-9222-watchdog" | head   # 看 watchdog 守护是否 load
```

主 Chrome 带 9222 启动 + watchdog 已 load 即安装成功。

Codex 用户额外确认 skill 软链：

```bash
ls -l ~/.agents/skills/web-tier ~/.agents/skills/opencli-web
```
