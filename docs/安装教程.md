# 安装教程（中文详版）

本文覆盖：**环境依赖** → **下载与安放位置** → **需按自己机器修改的地址** → **预热** → **验证**。

> 仅支持 **macOS**。Tier 3 用 launchd 守护独立 Chrome，依赖 macOS 特有机制。

---

## 一、环境依赖

### 1.1 必需依赖（不装跑不起来）

| 依赖 | 要求 | 检查命令 | 安装方式 |
|---|---|---|---|
| **macOS** | —— | `sw_vers` | 系统 |
| **Claude Code** | 最新版 | —— | 宿主环境，本 SOP 是它的 skill |
| **Node.js** | **≥ 22**（helper.mjs 用原生 `WebSocket`，22 起才默认开启） | `node -v` | [nodejs.org](https://nodejs.org) / `brew install node` / nvm |
| **npm** | 随 Node 自带 | `npm -v` | —— |
| **Google Chrome** | 任意近版 | `ls "/Applications/Google Chrome.app"` | [google.com/chrome](https://www.google.com/chrome/) |

### 1.2 可选依赖（按用到的功能装）

| 依赖 | 用途 | 不装的后果 | 安装方式 |
|---|---|---|---|
| **web-access 插件** | Tier 2 路由（主 Chrome 9222） | 只剩 Tier 1/3，路由表 Tier 2 分支缺位 | 见下方「二、3」 |
| **lark-cli** | `helper.mjs alert` 发飞书 IM 告警 | 告警只写本地日志 `~/.web-tier/alerts.log`，不报错 | 你的飞书 CLI 工具 |
| **jq** | 运行 `references/site-patterns/` 里的抓取示例脚本 | 示例脚本里的 `jq` 行会失败（自己改写即可） | `brew install jq` |
| **Tailscale** | 远程 SSH 救回登录态（SKILL.md 策略 SSH） | 纯本机使用用不到 | [tailscale.com](https://tailscale.com) |

### 1.3 系统自带（macOS 默认有，无需安装）

`curl`、`lsof`、`sqlite3`、`perl` —— launch.sh / health.sh / helper.mjs / opencli 超时护栏会用到，macOS 默认都带。

### 1.4 自动安装（`install.sh` 负责，无需手动）

| 依赖 | 说明 |
|---|---|
| **@jackwener/opencli** | 锁定 **1.7.22**。`install.sh` 跑 `npm ci` 装进 `skills/opencli-web/node_modules/`（该目录不入库，从 `package-lock.json` 重建） |

## 二、下载与安放位置

### 1. 下载本仓库

放哪都行（仓库会被 `install.sh` 软链到 skill 目录，不要装好后再移动它）：

```bash
git clone <本仓库地址> ~/web-tier-sop
cd ~/web-tier-sop
```

### 2. 安装本仓的两个 skill —— 安放到 `~/.claude/skills/`

**方式 A：自动（推荐）**

```bash
./install.sh
```

会软链 `skills/web-tier`、`skills/opencli-web` 到 `~/.claude/skills/`，并 `npm ci` 装 opencli。

**方式 B：手动**

```bash
# skill 必须位于 ~/.claude/skills/<skill名>/，Claude Code 才扫描得到
ln -s ~/web-tier-sop/skills/web-tier     ~/.claude/skills/web-tier
ln -s ~/web-tier-sop/skills/opencli-web  ~/.claude/skills/opencli-web
# opencli 依赖
cd ~/web-tier-sop/skills/opencli-web && npm ci
# runtime 脚本
mkdir -p ~/.web-tier
cp ~/web-tier-sop/runtime/launch.sh ~/web-tier-sop/runtime/health.sh ~/.web-tier/
chmod +x ~/.web-tier/launch.sh ~/.web-tier/health.sh
# launchd plist：把模板里的 __HOME__ 换成你的家目录
sed "s|__HOME__|$HOME|g" \
  ~/web-tier-sop/runtime/com.user.web-tier-chrome.plist.template \
  > ~/Library/LaunchAgents/com.user.web-tier-chrome.plist
```

> **关键**：skill 的识别目录固定是 `~/.claude/skills/<skill名>/SKILL.md`，不能放别处。

### 3. 下载 web-access 插件（Tier 2，第三方，可选）

`web-access` 是第三方插件（作者「一泽 Eze」），**不在本仓**，需单独下载。

**方式 A：Claude Code 插件命令（推荐）**

在 Claude Code 里执行：

```
/plugin marketplace add https://github.com/eze-is/web-access.git
/plugin install web-access
```

装好后它位于 `~/.claude/plugins/cache/web-access/`，由 Claude Code 自动管理，**无需手动安放**。

**方式 B：手动 git clone**

```bash
git clone https://github.com/eze-is/web-access.git
# 之后按 web-access 仓库自己的 README 安放
```

> **关于「更改地址」**：上面的 marketplace 地址 `https://github.com/eze-is/web-access.git` 是当前来源。
> 若原作者迁移仓库换了地址，把这个 URL 换成新地址即可，其余步骤不变。
>
> 只用 Tier 1/3，不装 `web-access` 也能跑 —— 只是路由表少了 Tier 2 分支。

## 三、需按自己机器修改的地址 / 配置

| 配置项 | 默认值 | 何时需要改 | 改哪里 |
|---|---|---|---|
| 家目录路径 | `install.sh` 自动渲染 | **仅手动安装时** —— 把 plist 模板的 `__HOME__` 换成 `$HOME` | `~/Library/LaunchAgents/com.user.web-tier-chrome.plist` |
| Chrome 程序路径 | `/Applications/Google Chrome.app/...` | Chrome 装在非标准位置 | `runtime/launch.sh` 的 `CHROME=` + plist 模板第一个 `<string>` |
| 独立 Chrome 端口 | `9223` | 端口被占用（少见） | `helper.mjs` `CDP_PORT` + `launch.sh` `PORT` + `health.sh` `PORT` + plist `--remote-debugging-port`（**多处，一般别动**） |
| 飞书告警目标 | 空（仅写本地日志） | 想收飞书 IM 告警 | 环境变量 `export WEB_TIER_ALERT_OPEN_ID=ou_xxxx`（用 `lark-cli auth list` 拿你的 open_id） |
| web-access 来源地址 | `https://github.com/eze-is/web-access.git` | 原作者迁仓 | 上方「二、3」的 marketplace 命令 |

## 四、预热独立 Chrome（必须在物理机前操作，不能远程 SSH）

预热会弹出 Chrome 窗口，远程 SSH 看不到也点不了，所以**必须坐在电脑前**做一次：

```bash
~/.web-tier/launch.sh
```

1. 弹出的 Chrome 窗口里，访问要登录的站点（如 `https://x.com/login`）用主号登录
2. `chrome://extensions/` 关掉所有扩展；`chrome://settings/passwords` 关掉密码保存
3. `Cmd+Q` 完全退出这个 Chrome
4. 交给 launchd 守护：

```bash
launchctl load ~/Library/LaunchAgents/com.user.web-tier-chrome.plist
```

完整预热细节见 [`../skills/web-tier/README.md`](../skills/web-tier/README.md)。

## 六、验证安装

```bash
~/.web-tier/health.sh                                  # 5 项健康诊断，应全 ✓
node ~/.claude/skills/web-tier/helper.mjs version       # 应返回 Chrome 版本号
```

`health.sh` 全部通过即安装成功。任一项失败，按它打印的 `→` 提示恢复。
